<task type="human">
  <name>Task 2: Benchmark GLM-OCR on actual 1970s newspaper scans</name>
  <files>validation/ocr-test.md</files>
  <read_first>
    - .planning/phases/01-foundation-validation/01-RESEARCH.md (GLM-OCR section — ollama.generate() API, base64 image input, prompt template)
    - src/mouse_research/config.py (AppConfig.ocr.ollama_url)
  </read_first>
  <action>
1. Ensure Ollama is running with GLM-OCR model:
   ```bash
   ollama serve &  # if not already running
   ollama pull glm-ocr  # if not already pulled
   ```

2. Find 3+ actual 1970s Gettysburg Times / York Daily Record newspaper scan images from Newspapers.com. These should be real scans Chris would process — multi-column layout, typical 1970s-80s print quality.

3. For each scan, run GLM-OCR via the Ollama Python SDK:
   ```python
   import ollama
   import base64
   from pathlib import Path

   image_data = base64.b64encode(Path("scan.jpg").read_bytes()).decode()
   response = ollama.generate(
       model="glm-ocr",
       prompt="Extract all text from this newspaper page image. Preserve the layout structure including headlines, subheadlines, bylines, and body text. Output as clean Markdown. If text is illegible, mark it as [illegible]. Do not guess or fabricate text that cannot be read.",
       images=[image_data],
   )
   print(response["response"])
   ```

4. For each scan, manually compare OCR output against the actual text visible in the image. Estimate Character Error Rate (CER): count character-level errors / total characters.

5. Document results in validation/ocr-test.md:
   - Image source (newspaper, date, description)
   - OCR output (full text)
   - Manually estimated CER
   - Did [illegible] markers appear appropriately?
   - Overall quality assessment: usable for searching/quoting?
   - Pass/fail verdict (pass if CER < 10% on at least 2/3 scans)

If GLM-OCR fails badly: document whether Tesseract with OpenCV preprocessing performs better, and whether the OCR strategy needs revision for Phase 2.
  </action>

# Milestone v1.0: MOUSE Research Pipeline

**Status:** SHIPPED 2026-04-02
**Phases:** 1-4
**Total Plans:** 16

## Overview

Four phases deliver the complete newspaper research pipeline for the MOUSE documentary project. Phase 1 resolves critical unknowns (newspapers-com-scraper and GLM-OCR accuracy on 1970s microfilm). Phase 2 builds the core single-URL archive pipeline. Phase 3 adds bulk search and batch archiving. Phase 4 adds the research graph (People/Source notes, master index).

## Milestone Summary

**Key Decisions:**
- Python as primary language, Node.js only for newspapers-com-scraper subprocess
- GLM-OCR via Ollama (primary), Tesseract (fallback) — both local, no cloud APIs
- Playwright storage_state for cookie management (launch_persistent_context has bug #36139)
- 500px max image dimension for GLM-OCR (GGML crash threshold)
- Append-only graph updates (never overwrite user content in People/Source notes)
- Person detection via explicit --person flag only (no NER/AI extraction)

**Issues Resolved:**
- Playwright Chromium macOS launch issues — solved with channel="chrome"
- GLM-OCR crashes on images > 500px — solved with preprocessing resize
- GLM-OCR hallucinates on full-page images — solved with article crop before OCR
- pydantic-settings yaml_file binding at class definition — solved with TestConfig subclass

**Tech Debt (from audit):**
- check_auth() defined but never called as pre-flight check (Cloudflare detection provides mid-run fallback)
- retry-failures counts skipped URLs as archived in summary display (cosmetic)
- 4 human verification items deferred to live testing (Phase 2)

**Stats:**
- 2,670 LOC source + 767 LOC tests = 3,437 total
- 14 Python modules, 65 tests
- Timeline: 2 days (2026-04-01 → 2026-04-02)
